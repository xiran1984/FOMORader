# FOMORader RULES（宪法）- v1.2

> 版本历史：v1.0 初稿 → v1.1 补充开发约束 → v1.2 数据源扩展为 X + RSS

## 核心原则
- FOMORader 用于采集、评估、筛选、总结、自动推送、预测趋势（短期快速增长）
- 打分结果必须缓存：同一个 URL + 同一个 LLM 模型 + 同一个 RULES 版本，只计算一次
- 不自动调整 RULES 权重（用户反馈仅记录，不影响当前运行版本）
- 用户需自行承担因 RULES 过时导致的质量判断偏差风险
- 数据源：X（中文高质量推文）+ 国际 AI RSS 源，统一存入 hotspots_raw.json
- 支持多 LLM 切换，但默认优先 Qwen（中文效果好）

## 高质量信息定义（固定权重，暂不自动更新）
总分 = 0.15 × 平台分 + 0.35 × 创作者分 + 0.50 × 内容分

### 平台分（0.15）
- X 中文高质量推文（min_faves:50, min_retweets:10）
- 国际 AI 权威 RSS 源（OpenAI/Anthropic/DeepMind 等）

### 创作者分（0.35）
- 账号权威性（粉丝量、认证状态）
- 历史内容质量
- 领域相关度

### 内容分（0.50）
- 原创性：非转发、非搬运
- 准确性：信息可验证
- 深度：有分析而非纯描述
- 时效性：发布时间权重

## 冷启动策略
- 系统首次启动时，从 data/hotspots_raw.json 加载最近一个月的热点作为种子数据
- 用户可手动补充 URL，逐步积累缓存

## 开发约束
- 禁止任何代理、VPN、镜像相关代码
- 禁止生成敏感政治内容示例
- LLM prompt 必须包含"严格遵守 FOMORader RULES 权重打分"
- 输出摘要必须控制在 200 字以内
- 禁止修改 SQLite 表结构（hotspots / scores / cache）除非同步更新 ARCHITECTURE.md
- 禁止在未读 ARCHITECTURE.md 的情况下修改数据流相关代码
